# CheburMail — Требования v1

## Требования v1

### Управление аккаунтами
- [ ] **ACCT-01**: Пользователь может настроить учётные данные Yandex Mail (IMAP/SMTP: сервер, порт, email, пароль приложения)
- [ ] **ACCT-02**: Пользователь может настроить учётные данные Mail.ru (IMAP/SMTP)
- [ ] **ACCT-03**: Приложение показывает пошаговую инструкцию создания пароля приложения на Yandex/Mail.ru
- [ ] **ACCT-04**: Пользователь может добавить несколько email-аккаунтов для увеличения пропускной способности
- [ ] **ACCT-05**: Приложение проверяет IMAP/SMTP подключение при настройке (тестовый логин)

### Криптография
- [ ] **CRYPT-01**: Приложение генерирует X25519 ключевую пару при первом запуске через libsodium randombytes_buf()
- [ ] **CRYPT-02**: Приватный ключ хранится безопасно через Tink + Android Keystore
- [ ] **CRYPT-03**: Пользователь может показать QR-код с публичным ключом + email
- [ ] **CRYPT-04**: Пользователь может отсканировать QR-код контакта для добавления его публичного ключа
- [ ] **CRYPT-05**: Сообщения шифруются через crypto_box (X25519 + XSalsa20-Poly1305, аутентифицированное шифрование)
- [ ] **CRYPT-06**: Уникальный nonce генерируется для каждого сообщения (24 байта через randombytes_buf)
- [ ] **CRYPT-07**: Пользователь может посмотреть safety number (отпечаток) контакта для верификации личности
- [ ] **CRYPT-08**: Пользователь может экспортировать зашифрованную резервную копию ключей для переноса на другое устройство
- [ ] **CRYPT-09**: Пользователь может импортировать резервную копию ключей на новом устройстве

### Сообщения
- [ ] **MSG-01**: Пользователь может отправить зашифрованное текстовое сообщение контакту (1-на-1)
- [ ] **MSG-02**: Пользователь может получить и расшифровать текстовые сообщения от контактов
- [ ] **MSG-03**: Сообщения доставляются через SMTP, закодированы Base64 в теле email с MIME-типом application/x-cheburmail
- [ ] **MSG-04**: Формат темы email: CM/1/<chat-id>/<msg-uuid> для маршрутизации и дедупликации
- [ ] **MSG-05**: Дедупликация входящих сообщений по UUID
- [ ] **MSG-06**: Пользователь может создать групповой чат (до 10 участников)
- [ ] **MSG-07**: Групповые сообщения рассылаются fan-out (отправитель шифрует индивидуально для каждого участника через crypto_box)
- [ ] **MSG-08**: Приглашение в группу через управляющее сообщение с раздачей публичных ключей участников
- [ ] **MSG-09**: Очередь offline-сообщений — сообщения ставятся в очередь в Room при отсутствии сети, отправляются при восстановлении
- [ ] **MSG-10**: Подтверждение доставки — получатель отправляет зашифрованный ACK-email обратно отправителю
- [ ] **MSG-11**: Исчезающие сообщения — пользователь может установить таймер автоудаления для чата (локальное исполнение)

### Транспорт
- [ ] **TRNS-01**: IMAP IDLE через foreground service для доставки сообщений в реальном времени (<5 сек)
- [ ] **TRNS-02**: WorkManager периодическая синхронизация каждые 15 мин как fallback (устойчив к Doze)
- [ ] **TRNS-03**: Выделенная IMAP-папка "CheburMail" с серверным фильтром для входящих сообщений
- [ ] **TRNS-04**: Очистка обработанных email через 7 дней для предотвращения исчерпания квоты
- [ ] **TRNS-05**: Экспоненциальный backoff при ошибках SMTP/IMAP (максимум 5 попыток)
- [ ] **TRNS-06**: Ротация multi-account при отправке для распределения нагрузки rate limit

### Интерфейс
- [ ] **UI-01**: Экран списка чатов с превью последнего сообщения
- [ ] **UI-02**: Экран чата с историей сообщений, полем ввода, кнопкой отправки
- [ ] **UI-03**: Управление контактами — добавление (через QR), просмотр, удаление
- [ ] **UI-04**: Мастер настройки аккаунта с пресетами Yandex/Mail.ru
- [ ] **UI-05**: Экран настроек (аккаунты, уведомления, параметры исчезающих сообщений)
- [ ] **UI-06**: Визуальный индикатор статуса сообщения (отправка/отправлено/доставлено/ошибка)
- [ ] **UI-07**: Уведомление о новом сообщении (в том числе из фоновой синхронизации)

### Хранение данных
- [ ] **STOR-01**: Room-база данных для истории сообщений, контактов, метаданных чатов
- [ ] **STOR-02**: Таблица очереди отправки (статусы: QUEUED/SENDING/SENT/FAILED)
- [ ] **STOR-03**: Таблица контактов с публичными ключами, отпечатками, статусом доверия
- [ ] **STOR-04**: Приватный ключ хранится через DataStore + Tink (AES с поддержкой Keystore)

## Требования v2 (Отложены)

- Пересылка файлов/картинок (лимит email ~25МБ)
- Голосовые сообщения
- Push-уведомления (требуют серверную инфраструктуру)
- iOS / Desktop клиенты
- Продвинутый обход rate limit (батчинг, планирование отправки)
- Обфускация метаданных (покрывающий трафик, relay-ноды)
- Знакомство контактов (существующий пользователь представляет двух контактов без физической встречи)
- Стеганографический режим (скрытие шифрованных данных в обычном email)
- Реакции на сообщения
- Редактирование сообщений

## Вне скоупа

- **Облачный бэкап сообщений** — нарушает принцип E2E, утечка метаданных
- **Отчёты о прочтении** — утечка метаданных об активности пользователя
- **Индикатор набора текста** — невозможно через email, создаёт ложные ожидания
- **Статус онлайн** — нет сервера для отслеживания, ложное ощущение присутствия
- **Firebase/FCM** — требует Google-сервисы, могут быть заблокированы
- **Центральный сервер** — весь смысл в P2P через email-инфраструктуру
